Camada de Domínio

Núcleo Central do Negócio

A camada de Domínio é isolada de frameworks ou bancos de dados. Ela contém as regras de negócio cruciais da aplicação culinária, garantindo estabilidade e permitindo mudanças tecnológicas sem esforço.

  • Entidades Puras Modelagem do domínio (ex: receitas, ingredientes, menus) utilizando Pydantic para validação sem dependência direta de ORM (SQLAlchemy).
  • Value Objects & Enums Encapsulamento de atributos específicos (como categorias de prato: entrada, principal, sobremesa) que contêm suas próprias regras de validação.
  • Repository Ports (Interfaces) Interfaces puramente abstratas como RecipeRepositoryPort definem o contrato de persistência. A implementação fica na Infraestrutura. 
"""Entidades base do domínio LaCuisine (Pydantic BaseModel, sem ORM)."""
from pydantic import BaseModel
from app.domain.enums.recipe_enums import RecipeCategory

class RecipeBase(BaseModel):
    """Entidade base de receita — campos compartilhados."""
    title: str
    category: RecipeCategory
    yield_amount: float
    yield_unit: str

class IngredientBase(BaseModel):
    """Entidade base de ingrediente."""
    name: str

class RecipeIngredientBase(BaseModel):
    """Entidade base de ingrediente de receita (relação N:N)."""
    amount: float
    unit: str
    notes: str | None = None
"""Ports (interfaces) para repositórios do domínio."""
from abc import ABC, abstractmethod
from uuid import UUID

class RecipeRepositoryPort(ABC):
    """Interface abstrata para o repositório de receitas."""

    @abstractmethod
    async def create(self, recipe_data: dict) -> dict:
        """Cria uma nova receita no banco."""
        ...

    @abstractmethod
    async def get_by_id(self, recipe_id: UUID, user_id: UUID | None = None) -> dict | None:
        """Busca uma receita por ID, verificando ownership."""
        ...

    @abstractmethod
    async def list_all(self, user_id: UUID | None = None) -> list[dict]:
        """Lista receitas, opcionalmente filtrando por user_id."""
        ...
"""Enums de Domínio."""
from enum import Enum

class RecipeCategory(str, Enum):
    """Categorias válidas de receitas."""
    APPETIZER = "appetizer"
    MAIN = "main"
    DESSERT = "dessert"
    OTHER = "other"
Camada de Aplicação

Casos de Uso da Aplicação

A camada de Aplicação implementa a lógica do caso de uso (Use Cases) e coordena o fluxo de dados de e para o Domínio. Ela desconhece como a informação chega ou é salva.

  • Fluxos Orquestrados (Use Cases) Toda ação relevante do sistema é representada por um caso de uso específico, como CreateRecipeUseCase ou ListRecipesUseCase.
  • Injeção de Dependências de Interfaces O caso de uso recebe o RecipeRepositoryPort no construtor. A injeção garante desacoplamento e facilita o mock em testes.
  • DTOs & Validação Básica Validações estruturais e integridade dos parâmetros da requisição antes de executar a lógica de negócios central. 
"""Use cases para operações de receitas."""
from uuid import UUID
from app.domain.ports.repository_ports import RecipeRepositoryPort

class CreateRecipeUseCase:
    """Caso de uso: Criar receita."""

    def __init__(self, repository: RecipeRepositoryPort):
        # Injeção via construtor (Dependency Inversion)
        self.repository = repository

    async def execute(self, data: dict):
        """Executa a validação do DTO e persistência no repositório."""
        required_fields = {"title", "category", "yield_amount", "yield_unit"}
        if not required_fields.issubset(data.keys()):
            missing = required_fields - set(data.keys())
            raise ValueError(f"Campos obrigatórios ausentes: {missing}")

        return await self.repository.create(data)

Camada de Infraestrutura

Infraestrutura e Integrações

A camada de Infraestrutura é onde a tecnologia é definida. Aqui ficam as implementações concretas dos repositórios SQL, gerenciamento de rotas HTTP (FastAPI) e serviços externos.

  • Repositório SQLModel (SQLAlchemy Async) O RecipeRepository implementa o Port definido no Domínio, utilizando consultas assíncronas no banco de dados relacional.
  • Injeção de Dependência do FastAPI (Depends) Gerenciamento de escopo de requisição, injeção automática de sessões de banco e autenticação JWT de forma nativa e limpa.
  • Serviços de IA (Gemini API) & Storage Integrações de imagem para extração de dados e conversas culinárias estruturadas com IA por trás de portas abstratas. 
"""Implementação concreta do RecipeRepositoryPort usando SQLAlchemy."""
from uuid import UUID
from sqlalchemy.ext.asyncio import AsyncSession
from app.domain.ports.repository_ports import RecipeRepositoryPort
from app.infrastructure.database.models import RecipeModel

class RecipeRepository(RecipeRepositoryPort):
    def __init__(self, session: AsyncSession):
        self.session = session

    async def create(self, recipe_data: dict) -> RecipeModel:
        """Cria e commita uma nova receita no banco de dados."""
        recipe = RecipeModel(**recipe_data)
        self.session.add(recipe)
        await self.session.commit()
        await self.session.refresh(recipe)
        return recipe
"""FastAPI Dependencies para Injeção de Dependência."""
from fastapi import Depends
from sqlalchemy.ext.asyncio import AsyncSession
from app.infrastructure.database.recipe_repository import RecipeRepository

async def get_db_session():
    """Gera sessões assíncronas do banco."""
    async with _session_factory() as session:
        yield session

async def get_recipe_repository(
    session: AsyncSession = Depends(get_db_session)
) -> RecipeRepository:
    """Injeta a dependência concreta do repositório."""
    return RecipeRepository(session)

from google import genai

class GeminiAIService(AIServicePort):

                "Extract recipe as JSON..."
            ]

Camada de Apresentação

Apresentação e Frontend

  • Componentes Altamente Visuais Interfaces como o Copiloto de Culinária (Chat) usam Framer Motion para transições fluidas e animações de feedback do assistente.
  • Custom Hooks e Contexto O hook de autenticação (useAuth) gerencia sessões globais de forma transparente e redireciona rotas protegidas.

Estratégia de Testes

Garantia de Qualidade

A arquitetura limpa do LaCuisine promove uma cobertura de testes simples e eficiente. Ao desacoplar dependências de IO, é possível testar lógica e UI de forma veloz e isolada.

  • Testes Unitários com Pytest (Backend) Validação estrita das entidades e regras de validação Pydantic. Execução em milissegundos sem necessidade de levantar banco de dados.
  • Testes de Integração Banco de dados de testes transacionais com rollback a cada execução. Mocking isolado de APIs externas de IA como Gemini.
  • Vitest & React Testing Library (Frontend) Testes de UI focados no comportamento do usuário, verificando renderização correta de cards de receita e submissões com Axios. 
"""Testes unitários puramente isolados no Pytest."""
import pytest
from pydantic import ValidationError
from app.domain.entities.recipe import RecipeBase
from app.domain.enums.recipe_enums import RecipeCategory

def test_criacao_com_dados_validos():
    """Garante que a entidade inicializa corretamente com valores padrão."""
    recipe = RecipeBase(
        title="Risotto de Trufa",
        category="main",
        yield_amount=4.0,
        yield_unit="porções",
    )
    assert recipe.title == "Risotto de Trufa"
    assert recipe.category == RecipeCategory.MAIN

def test_categoria_invalida_levanta_erro():
    """Garante que categorias fora do enum levantam ValidationError."""
    with pytest.raises(ValidationError):
        RecipeBase(
            title="Risotto",
            category="nao_existe",
            yield_amount=4.0,
            yield_unit="porções",
        )
